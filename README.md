1. Архитектура и принципы работы
Бот построен на модульной архитектуре с использованием современных инструментов обработки естественного языка (NLP):

Интерфейс: Streamlit — обеспечивает легкий веб-интерфейс и управление состоянием чата.
LLM (Мозг): Llama 3.3 70B — запущена через Groq API для обеспечения сверхвысокой скорости генерации (до 300 токенов в сек).

Обработка документов: * PDF Loader: langchain_community (PyPDF) для парсинга документации.

Text Splitter: RecursiveCharacterTextSplitter для нарезки текста на смысловые чанки по 1500 символов.

Хранение источников: Локальная файловая структура в репозитории GitHub. Документы организованы по папкам версий: ./docs/v27.6/, ./docs/v26.0/ и т.д.
Принцип поиска: "keyword-based priority search". Бот анализирует запрос и ищет совпадения в чанках, отдавая приоритет файлам, содержащим в названии adminguide, operatorguide или implementguide.


2. Функции бота
Мультиверсионность: Возможность выбора конкретной версии MaxPatrol 10 в боковом меню. Контекст поиска ограничен только выбранной версией.
Верификация ответов: Под каждым ответом выводится раскрывающийся список (Expander) с прямыми цитатами из PDF, указанием имени файла и номера страницы.
Умное кэширование: Использование @st.cache_resource позволяет мгновенно переключаться между версиями без повторного сканирования документов.
Инструменты пользователя: Кнопка быстрого копирования ответа в буфер обмена и кнопка очистки истории чата.
Безопасность данных: Бот работает только с тем контекстом, который нашел в документации. Если информации нет, он не выдумывает ответ.
